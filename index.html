Menjadi Developer ? Sudah Cukupkah Ilmu kita ?

Menjadi developer yang handal tentu saja tidak mudah, semua developer handal memulai dari awal, memulai dari bermacam - macam error, tapi mereka tetap konsisten ngoding, sehingga mereka tau apa yang mereka sudah bisa dan apa yang mereka belum bisa, mereka terus menerus belajar dan belajar sehingga lebih baik lagi.

Mungkin kita merasa bahwa mereka cukup hanya dengan usaha sedikit saja, maka program nya berfungsi dengan baik, sedangkan kita harus mati - matian untuk membuat sebuah program, tapi percayalah, bahwa mereka juga pernah mengalami hal yang sama, yaitu mengalami error dan berusaha setengah mati hanya untuk membuat sebuah program sederhana

Stuck ketika ngoding merupakan bagian dari proses kita menuju developer yang lebih baik lagi, tentunya dengan error dan stuck ini kita akan mencari tahu bahwa kodingn kit itu salah, dan kita harus mencari cara untuk membetulkan kodingan tersebut
ketika kita mengikuti sebuah tutorial padahal sudah sesuai, tapi selalu saja ada error disana, mungkin bukan error dari sisi kodingan kita, bisa jadi itu karena environment nya yang salah, karena yang paling susah adalah mengatur environment nya

Ketika anda belajar ngoding, buatlah catatan, apa yang sudah anda bisa kerjakan dan apa yg belum, dan fokus pada hal tersebut
semakin banyak anda belajar, semakin handal anda menjadi seorang developer

Pengalaman Belajar ASP.NET, NODEJS dan GOLANG

Awal bulan februari 2018 merupakan awal belajar bahasa pemrograman baru setelah sekian lama tidak belajar coding semasa kuliah,
dimulai dengan pengenalan javascript, html dan css kemudian disusul dengan penggunaan bootstrap untuk bagian front-end,
setelah beberapa minggu berlalu barulah mulai masuk ke bagian back-end,

Belajar ASP.NET MVC

yg pertama yaitu belajar bahasa pemrograman C# (c sharp), belajar c# mulai dari awal sekali, membuat aplikasi c# melalui console application, setelah beberapa hari barulah belajar ASP.NET dengan pola MVC (Model View Controller), MVC berbeda dengan OOP, Kalo OOP lebih menekankan bagaimana sebuah kode program ditulis sebagai sebuah objek yang memiliki atribut2 dan fungsinya, maka MVC ini cakupannya diatas itu, yakni mengatur arsitektur sebuah aplikasi. Umumnya aplikasi yang dibangun dengan konsep MVC adalah aplikasi yang cukup besar, karena salah satu keuntungan dari MVC itu adalah kemudahan maintenance, dan pengembangan aplikasi tersebut.
Konsep MVC :
Secara sederhana konsep MVC terdiri dari tiga bagian yaitu bagian Model, bagian View dan bagian Controller. Didalam website dinamis setidaknya terdiri dari 3 hal yang paling pokok, yaitu basis data, logika aplikasi dan cara menampilkan halaman wesite. 3 hal tersebut direpresentasikan dengan MVC yaitu model untuk basis data, view untuk cara menampilkan halaman website dan controller untuk logika aplikasi.

  1. Model
    Merepresantiskan struktur data dari website yang bisa berupa basis data maupun data lain, misalnya dalam bentuk file teks atau file xml. Biasanya didalam model akan berisi class dan fungsi untuk mengambil, melakukan update dan menghapus data website. Karena sebuah website biasanya menggunakan basis data dalam menyimpan data maka bagian Model biasanya akan berhubungan dengan perintah-perintah query SQL. Model bisa dibilang khusus digunakan untuk melakukan koneksi ke basis data oleh karena itu logika-logika pemrograman yang berada didalam model juga harus yang berhubungan dengan basis data. Misalnya saja pemilihan kondisi tetapi untuk memilih melakukan query yang mana.
  2. View
    Merupakan informasi yang ditampilkan kepada pengunjung website. Sebisa mungkin didalam View tidak berisi logika-logika kode tetapi hanya berisi variabel-variabel yang berisi data yang siap ditampilkan. View bisa dibilang adalah halaman website yang dibuat menggunakan HTML dengan bantuan CSS atau JavaScript. Didalam view jangan pernah ada kode untuk melakukan koneksi ke basis data. View hanya dikhususkan untuk menampilkan data-data hasil dari model dan controller.
  3. Controller
    Controller merupakan penghubung antara Model dan View. Didalam Controller inilah terdapat class dan fungsi-fungsi yang memproses permintaan dari View kedalam struktur data didalam Model. Controller juga tidak boleh berisi kode untuk mengakses basis data. Tugas controller adalah menyediakan berbagai variabel yang akan ditampilkan di view, memanggil model untuk melakukan akses ke basis data, menyediakan penanganan error, mengerjakan proses logika dari aplikasi serta melakukan validasi atau cek terhadap inputpenggunaan MVC sendiri mempermudah dalam pengelolaan aplikasi sehingga ketika error di suatu bagian, kita tinggal cari kesalahannya di bagian kodingan tersebut

ASP.NET MVC

ASP.NET mendukung tiga besar pengembangan View : Halaman Web, formulir Web dan MVC (Model View Controller). Framework ASP.NET MVC adalah Framework lightweight( kerja ringan), sangat diuji presentasi yang terintegrasi dengan fitur ASP.NET yang ada, seperti halaman master, otentikasi, dll. Dalam .NET, kerangka kerja ini didefinisikan dalam Majelis System.Web.Mvc. Versi terbaru dari framework kerja MVC adalah 5.0. Kami menggunakan Visual Studio untuk membuat ASP.NET MVC aplikasi yang dapat ditambahkan sebagai template dalam Visual Studio.

ASP.NET with C# Framework . NET adalah suatu komponen Windows yang terintegrasi yang dibuat dengan tujuan untuk mensupport pengembangan berbagai macam jenis aplikasi serta untuk dapat menjalankan berbagai macam aplikasi generasi mendatang termasuk pengembangan aplikasi Web Services XML
Fitur ASP.NET MVC
ASP.NET MVC menyediakan fitur berikut:

  1. Ideal untuk mengembangkan aplikasi kompleks
  2. Menyediakan framework extensible dan pluggable yang dapat dengan mudah diganti dan disesuaikan. Misalnya, jika Anda tidak ingin menggunakan built-in Razor atau ASPX view engine , maka Anda dapat menggunakan engine pihak ketiga view lain atau bahkan menyesuaikan yang sudah ada.
  3. Memanfaatkan desain berbasis komponen aplikasi dengan logis membagi menjadi komponen Model, View dan Controller. Hal ini memungkinkan para pengembang untuk mengelola kompleksitas proyek besar-besaran dan bekerja pada komponen individual.
  4. Struktur MVC meningkatkan pembangunan berbasis tes dan testability aplikasi karena semua komponen yang dapat dirancang dengan antarmuka berbasis dan diuji menggunakan mock objek. Maka ASP.NET MVC Framework sangat ideal untuk proyek-proyek dengan tim besar pengembang web.
  5. Mendukung semua yang ada luas ASP.NET fungsionalitas seperti otorisasi dan otentikasi, Master halaman, Data Binding, kontrol pengguna, keanggotaan, ASP.NET Routing, dll.
  6. Tidak menggunakan konsep View State (yang hadir dalam ASP.NET). Ini membantu dalam membangun aplikasi yang ringan dan memberikan kontrol penuh untuk para pengembang.

Dengan demikian, Anda dapat mempertimbangkan MVC Framework sebagai framework besar dibangun di atas ASP.NET menyediakan set besar menambahkan fungsionalitas dengan fokus pada pembangunan berbasis komponen dan pengujian

Belajar NODEJS

setelah beberapa waktu belajar ASP.NET, kemudian dilanjut dengan belajar NODEJS, sebelum belajar NODEJS, saya belajar MONGODB terlebih dahulu, karena NODEJS dengan MONGODB bekerja dengan baik, ini bukan berarti NODEJS tidak bisa dipasang dengan MYSQL atau yg lain, hanya saja NODEJS dan MONGODB itu seperti pasangan serasi, saya belajar nodejs mulai dari awal sekali, belajar menggunakan npm, menginstall library yg dibutuhkan kedalam project, di nodejs ini saya belajar membuat routing, contoh real nya yaitu bila kita masuk ke suatu URl tertentu, akan dilempar kemana kah si URL tersebut, kemudian saya juga belajar membuat middleware, apa itu middlware, berikut penjelasannya

Tujuan utama layanan middleware adalah untuk membantu memecahkan interkoneksi beberapa aplikasi dan masalah interoperabilitas. Middleware sangat dibutuhkan untuk bermigrasi dari aplikasi mainframe ke aplikasi client/server dan juga untuk menyediakan komunikasi antar platform yang berbeda.
Perangkat lunak ini terdiri dari serangkaian pelayanan yang mengizinkan bermacam-macam proses berjalan dalam satu atau lebih mesin untuk dapat saling berinteraksi satu sama yang lainnya. Lambat laun teknologi ini menyediakan kemampuan interoperabilitas yang mendukung pada perpindahan ke arsitektur distribusi yang berhubungan, yang biasanya sering digunakan untuk mendukung dan menyederhanakan kerumitan, aplikasi terdistribusi. Termasuk didalamnya, web server, aplikasi server dan peralatan sama yang mendukung pengembangan dan pengantaran aplikasi. Middleware secara khusus menjadi bagian dari teknologi informasi modern berbasis XML, SOAP, web service dan pelayanan berbasis arsitektur. Middleware berada diantara aplikasi perangkat lunak yang mungkin bekerja pada system operasi yang berbeda. Middleware serupa dengan middle layer dari sebuah tiga baris sistem arsitektur tunggal, kecuali usahanya melewati bermacam-macam system atau aplikasi.
Middleware berada diantara lapisan aplikasi (application layer) dan lapisan data dari sebuah arsitektur layer-layer TCP/IP. Middleware bisa juga disebut protokol.

Pelayanan middleware menyediakan banyak set fungsi dari aplikasi antarmuka pemogramanan yang mengizinkan sebuah aplikasi untuk:

  1. Menemukan tempat melewati jaringan secara transparan sehingga dapat menyediakan interaksi dengan service atau aplikasi lainnya.
  2. Mandiri dari service jaringan.
  3. Dapat dipercaya dan selalu tersedia.

Middleware menawarkan beberapa keuntungan unik dari technologi untuk bisnis dan industri. Sebagai contoh, sistem database tradisional biasanya diletakan dalam lingkungan yang dekat dimana pengguna mengakses sistem menggunakan jaringan terbatas atau intranet. Dengan perkembangan fenomena dari World Wide Web, pengguna dapat mengakses database secara virtual dengan berbagai macam jenis akses dari belahan dunia manapun. Middleware mengalamatkan masalah dari berbagai level interoperbilitas diantara struktur database yang berbeda. Middleware memfasilitasi akses transparan untuk melegalkan sistem manajemen database (DBMS) atau aplikasi lewat sebuah web server tanpa memperhatikan karakteristik spesifik database.

Trend Pengembangan Web 2018

Pengembangan Web 2018

KEMAMPUAN DASAR


Ini mungkin yang selalu dibutuhkan dan menjadi dasar dalam pengembangan web terutama di bagian front-end yaitu Html, Css, dan Javascript, ketiga hal ini saling berkaitan untuk membangun yang lebih menarik

NODE JS


Node JS bukan hal baru dalam dunia back-end, node js mulai populer mulai tahun kemarin, tahun ini pun diprediksi bahwa node js akan semakin banyak digunakan di bidang web development

FRONT END FRAMEWORKS


Vue JS, React JS, Angular merupakan framework Java Script untuk mempermudah web development di bidang front-end, ketiganya ditahun 2018 diprediksi akan menjadi framework yang paling populer digunakan oleh para front-end developer

WEB COMPONENT/COMPILERS


Web components adalah kumpulan fitur pada web browser yang memungkinkan kita untuk membuat widget (atau yang lebih dikenal dengan istilah component) dalam pengembangan web. Layaknya konsep plugin pada wordpress atau CMS serupa, component ini siap dipakai ulang pada halaman HTML manapun tanpa memerlukan library atau framework tambahan.

STATIC PAGES GENERATOR


static pages generator merupakan aplikasi statis yang hanya berisi html, css, javascript namun sudah multi platform, static pages generator bisa berjalan dengan baik di platform apapun
static pages generator mempermudah kita dalam membuat web statis yang indah dengan menggunakan bahasa markdown

PROGRESSIVE WEB APPS


Berbeda dengan aplikasi hybrid, aplikasi hybrid lebih menggabungkan antara aplikasi native dan web. PWA ini sepenuhnya web namun pengguna bisa merasakan experience seperti menggunakan aplikasi native. Seperti bisa dijalankan offline atau koneksi buruk sekalipun, bisa mengirim notifikasi, bahkan menggunakan animasi yang smooth seperti aplikasi native. Aplikasinya pun tidak perlu diinstall, namun icon-nya bisa di pasang di home screen untuk memudahkan membuka aplikasi. PWA sepenuhnya mengandalkan browser

SERVERLESS


pengembangan web juga sudah mulai meninggalkan client-server secara konvensional, beberapa tahun kebelakang sudah mulai migrasi serverless / cloud.
Arsitektur serverless membuat kita dapat mengembangkan aplikasi web dengan konfigurasi seminimal mungkin. Less pada istilah serverless bukan berarti tidak sama sekali, tapi mempunyai arti dikurangi. Sehingga ketika kita ingin deploy aplikasi serverless kita, kita tidak perlu berinteraksi banyak melalui jalan seperti SSH atau remote connection lainnya

ARTIFICIAL INTELIIGENCE / BOT


tahun ini Artificial Intelligence, Machine Learning akan semakin hype lagi, perkembangan AI maju sangat pesat, AI mulai mengambil alih pekerjaan manusia, bahkan di prediksi dalam beberapa tahun ke depan banyak sektor pekerjaan manusia yang akan diambil alih oleh robot/AI

Panduan Sederhana Perintah Git

–INSTALASI–

Unduh Git Untuk OSX
Unduh Git Untuk WINDOWS
Unduh Git Untuk LINUX

–PERUBAHAN LOKAL–

git init

buat inisialisasi repositori git baru

–PERIKSA REPOSITORI–

git clone /path/to/repository

buat mengcopy local repositori

git clone username@host:/path/to/repository

buat mengcopy file dari server

–ALUR KERJA–

git add <filename>

untuk stagging file, yaitu pemilihan file yang akan benar - benar dimasukan ke
repositori kita 

git add *

untuk stagging semua file dalam satu folder

git commit -m "commit message"

untuk memasukan file ke dalam repositori kita (-m yaitu untuk komentar terhadap file yang akan dimasukan)

–MENGIRIM PERUBAHAN–

git push origin master

untuk memasukkan file-file atau direktori lokal hasil pekerjaan yang sudah dilakukan
setelah melakukan commit ke repositori kita di online (github contohnya)

git remote add origin <server>

untuk menghubungkan repositori lokasi dan repositori di server/remote

–PERCABANGAN–

git checkout -b feature_x

untuk membuat branch baru

git checkout master

untuk masuk ke branch master

git branch -d feature_x

untuk menghapus branch tertentu

git push origin <branch>

untuk memasukkan file-file atau direktori lokal hasil pekerjaan yang sudah dilakukan
setelah melakukan commit ke repositori kita di online (github contoh) secara spesifik,
yaitu branch mana yang akan di upload

–MEMPERBAHARUI DAN MENGGABUNGKAN–

git pull

untuk mengambil/meng-update hasil pekerjaan kita/team/hasil kolaborasi di server (github contohnya)

git merge

untuk menggabungkan antara branch master dan branch yang diinginkan ketika pekerjaan
di suatu branch telah dianggap selesai

git diff<source_branch> <target_branch>

untuk melihat detail apa saja yang berubah

–MENANDAI VERSI–

git tag 1.0.0 1b2e1d63ff

untuk menandai versi dari projek kita, berdasarkan kode dari perubahan tersebut

–MELIHAT LOG–

1. git log

untuk melihat semua perubahan yang terjadi di dalam repositori kita

2. git log --author=ilham

untuk melihat perubahan yang dilakukan oleh akun tertentu

3. git log --pretty=oneline

untuk melihat tiap perubahan dalam satu baris

git log --graph --oneline --decorate --all

untuk meilihat tiap perubahan dalam satu baris
tetapi lebih rapi dengan mengurangi kode hash-nya menjadi hanya 5 baris

git log --name-status

untuk melihat tiap perubahan dengan melihat siapa akun yang merubahnya beserta
status perubahannya

git log –help

membuka halaman web di browser perintah apa saja yang ada di git log

–PERUBAHAN LOKAL–

git checkout -- <filename>

untuk mengembalikan ke keadaan sebalumnya, ketika melakukan kesalahan/perubahan yang tidak diinginkan

git fetch origin

untuk mengambil data dari server

git reset --hard origin/master

untuk menghapus seluruh perubahan untuk kemudian dikembalikan ke tahap stagging
